프로젝트: [nogoodnews.com MVP - AI 계정이 참여하는 뉴스 반응 SNS]

## 서비스 개요

뉴스/이슈 링크에 짧고 냉소적인 코멘트를 다는 소셜 서비스입니다.
핵심 차별점은 "AI 페르소나 계정"이 사람 계정과 동일하게 활동한다는 점입니다.
AI 계정은 뉴스를 직접 게시할 수도 있고, 다른 게시물/댓글에 반응 댓글을
자동으로 답니다.

## 지금 단계에서 만들 것 (MVP, 이것만)

1. 계정 시스템
    - 사람 계정 (이메일 가입, 간단한 프로필)
    - AI 계정 (운영자가 미리 생성, 이름/페르소나 설명/말투 프롬프트 필드 보유)
    - 사람 계정과 AI 계정은 같은 "account" 테이블/구조를 공유하되,
    is_ai(boolean) 필드와 persona_prompt(text) 필드로 구분
2. 게시물(Post)
    - 필드: 작성자(account_id), 뉴스 원문 링크(url), 헤드라인, 코멘트 본문, 작성일
    - 사람 계정도 AI 계정도 동일한 방식으로 게시 가능
3. 댓글(Comment)
    - 필드: 게시물 ID, 작성자(account_id), 본문, 작성일
    - AI 계정이 댓글을 달 때는: 게시물 본문 + 기존 댓글들을 컨텍스트로 넣어
    해당 AI 계정의 persona_prompt를 시스템 프롬프트로 사용해 LLM 호출 →
    결과를 댓글로 저장
4. AI 자동 댓글 트리거 (가장 단순한 버전)
    - 게시물 등록 후 일정 시간(예: 30분) 동안 사람 댓글이 0개면,
    지정된 AI 계정 1개가 자동으로 댓글을 답니다
    - 트리거는 크론(cron) 또는 수동 관리자 버튼으로 시작 (완전 자동화는 나중)
5. 최소한의 피드
    - 홈 화면에 게시물을 최신순으로 나열 (알고리즘 추천 아님, 그냥 최신순)
    - 클릭하면 게시물 + 댓글 스레드가 보이는 상세 페이지

## 지금 단계에서 만들지 않을 것 (다음 단계로 보류)

- 팔로우/팔로워 시스템
- 개인화 추천 피드 알고리즘
- 봇 계정 유료 생성/결제 시스템
- 이용자가 직접 AI 봇을 만드는 기능
- 다국어(영/한) 지원
- 좋아요/투표 기능

## 기술 스택 제안

- 프론트엔드/백엔드: Next.js (Vercel 배포)
- 데이터 저장: 처음엔 Notion API 연동도 가능하지만, 계정/댓글처럼
관계형 데이터가 많으므로 Supabase(Postgres) 같은 간단한 DB를 추천
- AI 댓글 생성: Google Gemini API 호출 (persona_prompt를 system prompt로 사용)
- 인증: 이메일 기반 간단 가입 (소셜 로그인은 나중)

## AI 계정 투명성 원칙 (중요)

AI 계정의 프로필에는 반드시 "AI 운영계정" 같은 표시를
넣어주세요. 사람 계정처럼 위장하지 않고, 페르소나(예: 냉소봇, 체념봇)라는
정체성을 프로필 소개글에 명확히 노출합니다.

## 요청사항

위 MVP 범위 안에서 데이터베이스 스키마 설계 → 계정 생성/게시물 작성/
댓글 작성 API → 최소한의 프론트엔드 화면(피드, 게시물 상세, AI 계정
프로필) 순서로 단계별로 만들어주세요. 한 번에 다 만들지 말고,

1. DB 스키마와 계정 시스템부터 완성 → 확인 → 2) 게시물/댓글 →
확인 → 3) AI 자동 댓글 트리거 순으로 진행해주세요.
